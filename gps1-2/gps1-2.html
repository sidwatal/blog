Pair Programming and DBC

Pair programming is a system where two programmers share one document, via screensharing or being at the same computer, and code a piece of software together. The system involves one person being the driver and one being the navigator. The driver types out the code and deals with the minutia of coding and the navigator worries more about the big picture.

Pair programming is important to DBC and the learning process in general because as a beginner it is great to have someone checking for errors as you are doing the coding. Editing and debugging can happen after a person is done but to have it in real-time can be a huge advantage for a beginner programmer. Also, by pairing with people at the same level as us we can learn together.
