One huge similarity between the two(and for this I am immensely grateful) are that they both have arrays that function very similarly and where Ruby has hashes, JavaScript has objects. Ruby arrays and JavaScript arrays are constructed almost identically. They both start at index[0] and they both can hold different data types. This makes creating and storing data in them super easy and convenient and I for one have never met any data I haven't immediately stored in an array.

The same(almost the same) can be said about Ruby hashes and JavaScript objects. Like a hash in Ruby, a JS object is a data structure where the elements are paired in a key/value system. The syntax of creating them with curly braces is also identical.

Where Ruby and JS differ when it comes to data structures is on the methods that can be called on them. While Ruby has all sorts of iterators and enumerables that can be called on arrays and hashes, in JS you are basically limited to the two types of loops(for and while). This makes things a little bit complicated but not impossible. Custom functions can be written using these two loops to basically do anything that the Ruby iterators and enumerables like each and map can do. Here is an example of what the map function would look like if you wrote it in JavaScript:

function map(array, func) {
var newArray = [];

for (var i=0; i < array.length; i++) {
newArray.push(func(array[i]));
}

return newArray;
};

As you can see, with just a few lines of code we have created the map method and can pass any array and code block to it. Keeping these pieces of code in your back pocket will be a great help for writing code in the future.
